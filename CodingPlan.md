## 二、客户端（用户端）

### 2.1 营销页面

面向潜在用户，展示产品价值促成转化。

| 编号  | 页面     | 路由                       | 功能说明                    |
|-------|----------|----------------------------|-----------------------------|
| C-P01 | 落地页   | ``` / ```                  | 产品介绍、核心优势、模型展示  |
| C-P02 | 定价页   | ``` /pricing ```           | 套餐对比（Lite/Pro）、价格展示 |
| C-P03 | 文档首页 | ``` /docs ```              | API 文档入口、快速开始       |
| C-P04 | 对接指南 | ``` /docs/integrations ``` | 第三方工具配置教程          |

#### 落地页模块

| 模块      | 说明                                         |
|-----------|----------------------------------------------|
| Hero 区域 | 主标题、副标题、CTA 按钮                       |
| 模型展示  | 首批支持模型卡片（Kimi2.5、Qwen3.5、GLM5）       |
| 工具集成  | 支持的工具图标（Cursor、Claude Code、VSCode 等） |
| 价格预览  | Lite/Pro 套餐简要对比                        |
| 页脚      | 链接导航                                     |

## 四、核心业务流程

### 4.1 用户订阅流程

```
落地页 → 定价页 → [登录/注册] → 控制台 → 套餐订阅 → 支付 → 订阅成功
                                           │
                    ┌──────────────────────┼──────────────────────┐
                    │                      │                      │
                    ▼                      ▼                      ▼
              API Key 管理             用量统计               订阅管理
                    │                      │                      │
                    ▼                      │                      │
              生成 API Key ────────────────┼──────────────────────┤
                                           │                      │
                                           ▼                      ▼
                                    查看调用明细           升级/降级套餐
```
 
> \[登录/注册\] 为通用子模块，本文档不展开。

### 4.2 API 调用流程

```
用户请求
    │
    ▼
┌─────────────┐
│ 1. 接收请求  │  ← 兼容 OpenAI / Anthropic 格式
└──────┬──────┘
       │
       ▼
┌─────────────┐
│ 2. 验证 Key  │  ← 检查 Key 有效性、是否禁用
└──────┬──────┘
       │
       ▼
┌─────────────┐
│ 3. 检查额度  │  ← 查询用户剩余额度
└──────┬──────┘
       │
   额度不足? ──是──▶ 返回 429 / 提示购买加油包
       │ 否
       ▼
┌─────────────┐
│ 4. 路由模型  │  ← 根据 model 参数路由到对应模型
└──────┬──────┘
       │
       ▼
┌─────────────┐
│ 5. 调用模型  │  ← 转发请求到上游模型服务
└──────┬──────┘
       │
       ▼
┌─────────────┐
│ 6. 返回结果  │  ← 格式化响应，保持兼容性
└──────┬──────┘
       │
       ▼
┌─────────────┐
│ 7. 扣减额度  │  ← 根据实际用量扣减（或按 Prompt 计数）
└─────────────┘
```
 
### 4.3 第三方工具对接流程

```
用户获取 API Key
       │
       ▼
配置到第三方工具（Cursor/Claude Code/VSCode）
       │
       ▼
工具发起请求 ──▶ Codingplan API ──▶ 模型服务
       │
       ▼
返回结果给工具
```

## 五、首批支持范围

### 5.1 首批支持模型

| 模型        | 上下文窗口 | 特点         | 适用场景            |
|-------------|------------|--------------|---------------------|
| **Kimi2.5** | 200K       | 长文本处理强 | 代码审查、长文档分析 |
| **Qwen3.5** | 128K       | 中文理解优秀 | 中文编程、注释生成   |
| **GLM5**    | 128K       | 代码能力强   | 代码生成、算法实现   |

### 5.2 首批支持工具

| 工具               | 对接方式       |
|--------------------|----------------|
| **Cursor**         | OpenAI 兼容    |
| **Claude Code**    | cc-switch 配置 |
| **VSCode + Cline** | OpenAI 兼容    |
| **OpenCode**       | OpenAI 兼容    |
| **OpenClaw**       | OpenAI 兼容    |

### 5.3 兼容接口

| 接口                    | 路径                                | 说明            |
|-------------------------|-------------------------------------|-----------------|
| OpenAI Chat Completions | ``` POST /v1/chat/completions ```   | 标准兼容接口    |
| Anthropic Messages      | ``` POST /anthropic/v1/messages ``` | Claude 兼容接口 |
